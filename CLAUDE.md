## What this is

A C# client SDK for the [MangaUpdates API](https://api.mangaupdates.com/), distributed as the `MangaUpdates` NuGet package. **Almost all of the code is auto-generated by [OpenAPI Generator](https://openapi-generator.tech)** (csharp generator) from an OpenAPI spec — treat `src/MangaUpdates/**` as generated output, not hand-written source (see "Generated code" below before editing anything there).

## Build / test commands

```bash
dotnet build -c Release                     # build the solution
dotnet test  -c Release                      # run all tests (xUnit)
dotnet pack src/MangaUpdates/MangaUpdates.csproj -o ../../output -c Release --no-build
```

Run a single test:
```bash
dotnet test --filter "FullyQualifiedName~SeriesApiTests.RetrieveSeriesTest"
dotnet test --filter "DisplayName~RetrieveSeries"
```

Target framework is `net6.0` for both the library and the test project.

## Regenerating the client

The client is regenerated from the OpenAPI spec via the openapi-generator CLI:
```bash
npx @openapitools/openapi-generator-cli generate -c openapi-dotnet.yaml
```
- Generator config: `openapi-dotnet.yaml` (generatorName `csharp`, `packageName: MangaUpdates`, `targetFramework: net6.0`, and a `integer=long` type mapping — this is why IDs are `long`, not `int`).
- Pinned generator version: `6.6.0` (`openapitools.json`).
- Input spec: `api-yaml/openapi.yaml`. Note `api/openapi.yaml` also exists and **differs** from `api-yaml/openapi.yaml`; `openapi-dotnet.yaml` points at `api-yaml/`, so that is the source of truth for generation.

### Generated code — read before editing `src/`

- Files under `src/MangaUpdates/` carry an OpenAPI Generator header and are **overwritten on regeneration**. Hand-edits there are lost unless the file is listed in `.openapi-generator-ignore` (currently that ignore file has no active entries — every generated file is fair game to be overwritten).
- There is a layout mismatch to be aware of: `openapi-dotnet.yaml` writes to `outputDir: generated/`, and `.github/workflows/release.yml` builds from a `generated/` working directory, but the committed, buildable code lives under `src/` (moved there per the latest commit, "moved to root"). If you regenerate, output lands in `generated/` and must be reconciled with `src/` — do not assume a plain regen produces a buildable tree in place.
- Known manual deviations from stock generation that must be preserved across a regen: the **Polly** dependency + `src/MangaUpdates/Client/RetryConfiguration.cs` (retry policies, not part of stock output) and the `System.Web` reference in `MangaUpdates.csproj`.

## Architecture

Standard OpenAPI-generator C# client shape, three namespaces:

- **`MangaUpdates.Api`** (`src/MangaUpdates/Api/`) — one class per API tag (e.g. `SeriesApi`, `MembersApi`, `ReleasesApi`, `GroupsApi`, ~17 total). Each API class implements paired sync/async interfaces (`I{Name}ApiSync` / `I{Name}ApiAsync`) and exposes each operation in two forms: a plain method returning the model, and a `...WithHttpInfo` variant returning `ApiResponse<T>` (status code + headers + data).
- **`MangaUpdates.Model`** (`src/MangaUpdates/Model/`, ~355 classes) — request/response DTOs. Versioned model names carry a `V1` suffix (e.g. `SeriesModelV1`, `ApiResponseV1`). Polymorphic types use `JsonSubTypes`.
- **`MangaUpdates.Client`** (`src/MangaUpdates/Client/`) — transport and config layer built on **RestSharp** + **Newtonsoft.Json**. Key types: `ApiClient` (`ISynchronousClient`/`IAsynchronousClient` impl), `Configuration`/`GlobalConfiguration`/`IReadableConfiguration` (base path, auth, timeouts), `ApiException`, `ApiResponse<T>`, `RequestOptions`, `Multimap`, `RetryConfiguration` (Polly).

Auth is bearer token; most GET endpoints are public and need no credentials.
